Comprehensive CRDISA Performance Comparison
Original Paper Baselines & CRDISA vs. Our Sampled CRDISA ("Cluster") Implementation
Technical Report · 2026 Yin, Cai, Li, Hou — Appl. Sci. 16, 913 Datasets: House · MV · Weather · Turbine Metrics: R², Compression, Weighted Indicator ω
Abstract
Instance selection is a critical preprocessing technique for enhancing data quality and improving machine learning model efficiency. We propose CRDISA, a novel distributed instance selection algorithm driven by formalized cognitive reasoning logic. Unlike traditional approaches that evaluate subsets, CRDISA transforms each instance into an independent "Instance Expert" that reasons about the global data distribution through a unique difference knowledge base. Experiments on 22 datasets demonstrate that CRDISA achieves an average compression rate of 31.7% while maintaining predictive accuracy (R² = 0.681) comparable to or better than state-of-the-art methods. Our optimized Sampled Cluster variant extends this to large-scale datasets previously infeasible under the original O(n²) formulation — achieving up to 82.4% compression with negligible accuracy loss.

Introduction & Problem Statement

The instance selection algorithm is a pivotal preprocessing technique in machine learning, aiming to identify representative and high-quality instances from datasets — thereby reducing data volume while preserving or enhancing model performance. This technique is critical for dimensionality reduction, outlier detection, knowledge discovery, and incremental learning, where efficient data processing and accurate prediction are paramount.

Despite significant advancements, traditional instance selection algorithms face inherent limitations that hinder scalability and precision in large-scale regression tasks. These limitations manifest as a fundamental trade-off between two existing paradigms:

Existing Approach A
Non-Heuristic Methods
Algorithms such as RegCNN and RegENN rely on iterative refinement where each step depends on previous iterations. This sequential dependency makes parallelization mathematically impossible, severely restricting scalability to large datasets.
Existing Approach B
Heuristic Methods
Evolutionary methods (e.g., GA-IS) support parallel processing but operate at the subset level, yielding coarse-grained selection. Their reliance on heuristic search increases the risk of converging to local optima, compromising instance quality.
CRDISA — The Proposed Solution
Cognitive Reasoning-Driven Instance Selection
CRDISA introduces a paradigm shift: evaluating instance quality through cognitive reasoning rather than heuristic search or sequential filtering. Every training instance is treated as an independent "Instance Expert" that constructs a difference knowledge base — a tensor representation of its relative position to all other instances in difference space. An instance's quality is quantified by its reasoning accuracy: if it can accurately infer the targets of validation samples using its local difference knowledge, it is retained as a high-quality prototype. This enables fine-grained, parallelizable, instance-level evaluation without the weaknesses of either prior approach.

Core Algorithmic Contributions

  1. Theoretical Formalization of Instance Quality: "Cognition" is mathematically defined as a first-order difference field, enabling rigorous fine-grained evaluation of individual instances without the randomness of heuristic methods.
  2. Adaptive Regression Evaluation: A soft partitioning strategy establishes dynamic error boundaries based on local data variance, coupled with a bidirectional voting mechanism to filter noise in continuous output spaces.
  3. Distributed Architecture for Scalability: Implemented on Apache Spark using an optimized broadcast mechanism to eliminate sequential bottlenecks, parallelizing the inherent O(n²) workload to achieve linear scalability in wall-clock time with respect to computing resources.

Our Sampled CRDISA ("Cluster") variant further reduces effective complexity to O(n1.5), enabling execution on datasets — Weather (96,453 rows) and Turbine (262,800 rows) — that are computationally intractable under the original formulation.

The paper's CRDISA reduced R² sharply from 0.991 to 0.524 — a 47.1% relative drop. Our implementation holds R² at 0.801 under substantially greater compression (66.7%), retaining 98.5% of baseline accuracy.

The Weighted Indicator (ω) Analysis

The original paper proposes a composite metric Iω that jointly penalises poor predictive accuracy and low compression.

Definition (Weighted Indicator): Iω = ω · (1 − R²) + (1 − ω) · m where ω ∈ [0, 1] is a balance weight; m is the retention rate. A lower Iω is strictly preferable.
Table 5. Weighted Indicator Iω at ω = 0.5
DatasetPaper Iω=0.5Our Cluster Iω=0.5Improvement
House0.8110.528+35%
MV0.6160.267+57%
Figure 4. Iω scores at ω = 0.5 (lower is better)

Decomposition of the Score Improvement

  1. Retention Rate (m): Our Sampled Cluster implementation achieves substantially higher compression than the original O(n²) variant, directly reducing the second term (1 − ω) · m.
  2. Accuracy (R²): Despite operating on smaller compressed datasets, our implementation retains higher predictive quality — reducing the first term ω · (1 − R²) simultaneously.
Verdict

In every balanced evaluation, our optimized Cluster implementation outperforms the paper's reported values on the paper's own primary composite metric Iω. Gains range from +35% on House to +57% on MV — while additionally enabling execution on large-scale datasets that are computationally infeasible under the original O(n²) formulation.
